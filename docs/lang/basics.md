# 先摸清砖块

> 代码：[`code/lang-basics/`](../../code/lang-basics/)　  
> 运行：在 `code/` 目录下执行 `cargo run -p lang-basics`

> 前置：[《环境与工具链》](../start/toolchain.md)（能 `cargo run`）。  
> Go 里 `int`、`string`、`bool` 你闭着眼都会用；Rust 的砖块更「咬文嚼字」—— 整数有多宽要说清、数值不帮你偷偷转型、没有零值概念。  
> 这一章把日常 90% 会碰到的基础类型过一遍，后面所有权、集合都踩在它们上头。

读完你应该能：**看见一个类型名就知道它大概多重、能不能拷贝、和 Go 谁对应**；并养成「该写类型标注就写、该 `as` 就显式转」的手感。

# 整数写明宽度

> Rust 没有「随平台变宽」的默认 `int`（那是 `isize`/`usize` 的事）。日常业务整数，优先想 `i32` / `i64` / `u64`。

| Rust | 含义 | Go 近似 |
| --- | --- | --- |
| `i8` … `i128` | 有符号，固定宽度 | `int8` …（Go 无 i128） |
| `u8` … `u128` | 无符号，固定宽度 | `uint8` … |
| `isize` / `usize` | 指针宽度（32/64 位机不同） | `int` / `uint`（平台相关） |
| `i32` | **最常见** 的有符号默认推断 | 常对应你随手写的 `int` |

```rust
let a: i32 = 42;
let b: u64 = 1_000_000;   // 下划线只为好看
let c = 255u8;            // 后缀定型：这是 u8
let hex = 0xff_u16;
let bin = 0b1010_u8;
```

对照 Go：Go 的 `int` 在 64 位机器上是 64 位，写库时还得小心端口。  
Rust 把「到底几位」写进类型名， **跨平台行为一致**，心智更简单。

`usize` 用在哪：索引、长度、`Vec::len()` 的返回值——「和内存地址同宽」的量。  
别用 `i32` 当索引去跟 `len()` 比大小，类型对不上还要转来转去。

# 浮点布尔和字符

> 三个小类型，坑都不大，但 `char` 和 Go 的 `byte`/`rune` 别混。

**浮点**：`f32` / `f64`。字面量 `3.14` 默认推断成 **`f64`**（不是 f32）。 对照 Go：`float64` 是主流，两边习惯接近。

**布尔**：只有 `true` / `false`。没有「非 0 即真」——`if 1` 直接编译不过。 对照 Go：Go 也不允许 `if 1`，这一点你们早就一致了。

**字符 `char`**：一个 Unicode **标量值**，在内存里固定 **4 字节**。 可以是 `'A'`，也可以是 `'🦀'`。它 **不是** UTF-8 里的一个字节。

```rust
let pi: f64 = 3.14159;
let ok = true;
let ch = '🦀';          // 一个 char，不是字符串
let ascii = 'A';
```

对照 Go：

| Go | Rust | 别搞混 |
| --- | --- | --- |
| `byte`（`uint8`） | `u8` | 原始字节 |
| `rune`（`int32`） | `char` | Unicode 码点（语义接近） |
| `string` 里按字节下标 | 见 [《字符串、数组与切片》](strings-slices.md) | Rust 字符串按字节索引可能切到字符中间 |

> 🔬 **为什么 char 是 4 字节？** Unicode 标量最大到 `U+10FFFF`，用 `u32` 能装下任意一个；  
> UTF-8 编码则是 1～4 个 `u8`。Rust 把「逻辑字符」和「磁盘上的字节序列」拆成两种类型，强迫你想清楚自己在处理哪一层。

# 推断与显式标注

> 多数时候编译器能猜对类型（像 Go 的 `:=`）。  
> 猜不出或想写清楚时，就在左边加 `: 类型`。

```rust
let x = 10;        // 整数默认 → i32
let y = 2.5;       // 浮点默认 → f64

// 空 Vec 还没有任何元素，猜不出装什么 → 必须标注（或稍后 push 一次让它推断）
let mut ids: Vec<i64> = Vec::new();
ids.push(7);

let n: i64 = 1;    // 左边标注：既帮编译器，也帮三个月后的自己
```

对照 Go：`var n int64 = 1` vs `n := int64(1)`。  
Rust 更常写在 `let` 右边靠推断， **在 API 边界、空集合、数值可能歧义时** 再显式标注——这是好品味，不是啰嗦。

# 数值从不偷偷转

> 这是 Go 转 Rust 的小震撼：`i32` 和 `i64` **不能直接相加**。  
> 没有 C/Go 那种「隐式加宽」。你必须亲口说怎么转。

```rust
let a: i32 = 10;
let b: i64 = 20;
// let sum = a + b;          // ❌ mismatched types
let sum = a as i64 + b;      // ✅ 显式：先把 a 加宽再加
```

**`as` 转换**：按目标类型的规则做截断或符号扩展， **总能写出来** —— 包括危险的 `300 as u8`（结果是 44）。编译器不拦你，责任在你。

更稳的路：

- **加宽、一定成功**：用 `.into()` / `From`（如 `i32` → `i64`）；
- **可能失败**：用 `try_into()` / `TryFrom`，拿到 `Result`，对照 Go 里自己检查范围。

```rust
let n: i32 = 5;
let wide: i64 = n.into();    // 安全加宽
```

对照 Go：`int32(x)` / `int64(x)` 也是显式的，但同「族」运算时常更松。  
Rust 把松的那扇门焊死了——短期别扭，长期少一串静默截断 bug。

# 元组捆异质数据

> 元组 `(T, U, …)`：几个类型可以不同的值，临时捆一手。  
> 对照 Go 的多返回值，但元组是 **一等值**，可以赋给变量、放进结构体。

```rust
let pair: (i32, &str) = (1, "hello");
println!("{}, {}", pair.0, pair.1);   // 按下标取

let (id, name) = pair;                // 解构拆包，≈ Go 的 a, b := f()
```

要点：

- 单元素元组必须写成 `(42,)`，光 `(42)` 只是括号表达式；
- 字段太多就别用元组了——改成 `struct`，名字比 `.0 .1` 好读；
- 函数「多返回值」在 Rust 里就是「返回一个元组」（或结构体）。

----

# 定长数组

> `[T; N]`：长度 `N` **写进类型**。`[i32; 3]` 和 `[i32; 4]` 是两种类型。  
> 对照 Go 的 `[3]int`（也是定长）；Go 里你更常用切片，Rust 日常主力则是后面的 `Vec`。

```rust
let nums: [i32; 3] = [10, 20, 30];
let zeros = [0i32; 5];        // 5 个 0
let first = nums[0];          // 越界 → panic（不是未定义行为）
```

和 Go 的差异：

| | Go `[3]int` | Rust `[i32; 3]` |
| --- | --- | --- |
| 传参 | 复制整块（或靠指针） | 元素是 `Copy` 则整体可拷贝 |
| 变长 | 改用 slice | 改用 `Vec<T>` / 切片 `&[T]` |
| 越界 | panic | panic |

数组适合「长度编译期就定死」的场景（小缓冲、查表）。  
要增长、要当函数参数传「任意长」——用 [《集合：Vec 与 HashMap》](collections.md) 和 [《字符串、数组与切片》](strings-slices.md)。

# 没有零值这回事

> Go：`var n int` 得 `0`，`var p *T` 得 `nil`，`var s string` 得 `""`。  
> Rust：**变量必须先初始化再使用**，没有「声明了但还是零」的窗口。

```rust
let n: i32;
// println!("{n}");   // ❌ 可能尚未初始化，编译器拒绝
let n = 0;            // ✅ 你得自己给初值（需要 0 就写 0）
```

「可能没有」用 `Option<T>`（[《Rust 语法底座》](../start/syntax-primer.md) 已扫过），不是靠零值假装有。  
这是全书错误处理、HashMap 取值的同一套哲学。

# unit 和下划线

> `()`（unit）：表示「没有有意义的返回值」，约等于 Go 里「不写返回类型」。  
> `_`：解构或赋值时说「这个位置我故意不要」。

```rust
fn log_hi() {                 // 等价于 -> ()
    println!("hi");
}

let (x, _, z) = (1, 2, 3);    // 只要两头
let _ = some_result();        // 显式丢弃，避免 unused 警告
```

`let _ = …` 和「根本不写」不同：前者表示「我知道有返回值，故意扔」；  
在必须处理 `Result` 的上下文里，乱扔会被 clippy/编译器盯——那是后话。

# 遮蔽不是可变

> 同名 `let` 可以再来一次，新绑定 **遮住** 旧的，类型都可以变。这叫 shadowing， **不是** `mut`。

```rust
let x = 5;
let x = x + 1;              // 新 x，值 6
let x = "现在是字符串";      // 合法：新变量，类型换成 &str
```

对照 Go：没有这种「同名换类型」的惯用写法；你要么新变量名，要么改设计。  
Rust 里常见于：先拿到 `&str`，再 `let s = s.to_string()` 变成拥有者——名字不用换。

和 `mut` 的分工：

- 要 **改同一个值**：`let mut x`，类型不变；
- 要 **变换成另一种形态**：再 `let x = …` 遮蔽。

# 三句话带走

1. **宽度写进类型名**，日常整数想 `i32`/`i64`/`u64`，长度用 `usize`。  
2. **数值不隐式转换**，`as` 能转但可能截断；安全加宽优先 `into`。  
3. **没有零值**，没有「非 0 即真」；不确定就用 `Option`。

# 附：本章生词表

- **标量类型**：整数、浮点、布尔、字符——单一值，不再拆分。  
- **`isize` / `usize`**：与指针同宽的有符号/无符号整数，多用于索引与长度。  
- **字面量后缀**：如 `255u8`、`1_000u64`，直接钉死字面量类型。  
- **`as` 转换**：强制类型转换，可能截断；不同于 `From`/`Into` 的「有实现才转」。  
- **`From` / `Into`**：标准转换 trait；`a.into()` 常等价于 `Target::from(a)`。  
- **`TryFrom` / `try_into`**：可能失败的转换，返回 `Result`。  
- **元组 `(T, U)`**：固定长度的异质组合，一等值。  
- **定长数组 `[T; N]`**：长度属于类型一部分的数组。  
- **unit `()`**：空类型/空值，表示「无信息」。  
- **遮蔽（shadowing）**：同名 `let` 重新绑定，遮住旧变量。  
- **零值**：Go 里类型未赋值时的默认值；Rust 无此概念。
